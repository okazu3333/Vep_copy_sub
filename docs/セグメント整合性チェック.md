# セグメント整合性チェック結果

## 📊 セグメント定義

### トーンダウン (forecast_tone_down)
- **説明**: 文面の勢いが落ち、温度感が下がっている予兆
- **検知ラベル**: ネガティブ表現／定型返信への偏り
- **アクション**: 背景ヒアリングと前向きな提案で温度感を回復
- **カテゴリ**: 予兆 (forecast)

### 催促 (occurrence_followup)
- **説明**: 回答期限の明示や強いリマインドが入っている状態
- **検知ラベル**: 「至急」「いつまで」などの催促フレーズ
- **アクション**: 状況共有と回答コミットで信頼低下を防ぐ
- **カテゴリ**: 発生 (occurrence)

## ❌ 整合性の問題点

### 1. `dummy-alert-002` (forecast_tone_down) - **不整合**

**現在の内容**:
- 件名: 「大型プラン追加の稟議支援」
- AI要約: 「大型拡張の稟議資料ドラフトを当日中に求められており、担当役員が強くフォローを期待しています」
- 検知理由: 「上層部から当日中の資料共有を要求」

**問題点**:
- 「当日中に求められている」「強くフォローを期待」は**催促**的な内容
- 「トーンダウン」（温度感が下がっている予兆）とは整合性が取れていない
- むしろ**`occurrence_followup`（催促）**に該当する内容

**修正案**:
- `primarySegment`を`occurrence_followup`に変更
- または、内容を「トーンダウン」に合うように変更（例: 返信が短くなった、定型返信が増えた、温度感が下がっている）

### 2. `dummy-alert-005` (occurrence_delay) - **セグメント不一致**

**現在の内容**:
- 件名: 「PoCレポート提出への強いフォロー」
- AI要約: 「PoC結果レポートの未提出が続き、顧客が正式契約への不安を表明しています」
- メール内容: 「明日17時が経営会議の締め切りとなりますので、それまでに正式版をご送付いただけますでしょうか」
- `primarySegment`: `occurrence_delay`

**問題点**:
- 内容は「催促」的な要素が強い（「明日17時が締め切り」「遅れる場合は理由とリカバリー案を併せて」）
- `occurrence_delay`（遅延）ではなく、**`occurrence_followup`（催促）**に該当する内容

**修正案**:
- `primarySegment`を`occurrence_followup`に変更
- `detectionReasons`を`SEGMENT_META.occurrence_followup.detectionLabel`に変更

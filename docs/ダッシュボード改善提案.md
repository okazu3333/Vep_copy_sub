# ダッシュボード改善提案（BagFlow参考）

## BagFlowダッシュボードの良い点

### 1. **レイアウト構造**
- 左サイドバー：ナビゲーション
- 上部ヘッダー：フィルター、保存ビュー、期間選択、最終更新時刻
- メインエリア：KPIカード → グラフ → ステータスカード → 最近の更新

### 2. **KPIカードの表示**
- 数値が大きく見やすい
- トレンド表示（+3.1%、-1.4%など）
- アイコンで視覚的に分かりやすい
- 目標値との比較（例：平均販売日数 ≤15日）

### 3. **グラフの配置**
- モデル別販売数：棒グラフで色分け
- 処理時間：時系列で目標線（15分）を表示
- 凡例が明確

### 4. **ステータスカード**
- 「要承認」バッジで注意喚起
- 数値と説明が明確
- アイコンで視覚的に識別

### 5. **最近の更新フィード**
- 時系列で表示
- ユーザー名、時間、ステータスボタン
- リアルタイム感

## 現在のダッシュボードとの比較

### 現在の課題
1. KPIカードがシンプルすぎる（トレンド表示なし）
2. グラフの配置が散らばっている
3. 最近の更新がない
4. ヘッダーにフィルター機能がない
5. 処理時間の可視化がない

### 改善提案

#### 1. KPIカードの改善
- トレンド表示を追加（前日比、前週比）
- 目標値との比較表示
- アイコンの改善

#### 2. レイアウトの改善
- 上部にKPIカードを横並び配置
- グラフを2列グリッドで配置
- 右側に「最近の更新」フィードを追加

#### 3. ヘッダーの追加
- 期間選択（過去7日、過去30日など）
- 保存ビュー機能
- 最終更新時刻表示

#### 4. ステータスカードの追加
- 要対応アラート数
- セグメント別の検知状況
- 緊急度の高いアラート

#### 5. 処理時間の可視化
- アラート検知から対応開始までの時間
- 対応開始から解決までの時間
- 目標時間との比較

# カード間余白改善案

## 現状のカード間余白

### 1. **KPIカード**
- `gap-4` (16px) - 4つのカード間

### 2. **営業/セグメント可視化セクション**
- `gap-6` (24px) - 2つのカード間（営業別・会社別アラート本数 と セグメント別アラート数）

### 3. **メンバー状況と最新更新セクション**
- `gap-6` (24px) - 2つのカード間（メンバー別アラート数グラフ と 最近の更新）

### 4. **案件状況セクション**
- `gap-6` (24px) - 2つのカード間（フォロー中の案件 と 緊急度の高いアラート）

### 5. **フォロー中の案件内のグリッド**
- `gap-3` (12px) - 案件カード間

## 問題点

1. **カード間の余白が不統一**
   - KPIカード: 16px
   - その他のカード: 24px
   - フォロー中の案件内: 12px

2. **カード内パディングとのバランス**
   - カード内パディング: `p-5` (20px)
   - カード間余白: 16px または 24px
   - カード内パディング（20px）とカード間余白（24px）の比率が不自然

3. **視覚的な階層が不明確**
   - 同じレベルのカードなのに余白が異なる

## 改善案

### 推奨設定（BagFlowスタイル参考）

1. **カード間余白の統一**
   - すべてのカード間: `gap-5` (20px) に統一
   - カード内パディング: `p-6` (24px) に変更
   - これにより、カード内パディング（24px）> カード間余白（20px）となり、視覚的な階層が明確になる

2. **フォロー中の案件内グリッド**
   - `gap-3` (12px) → `gap-4` (16px) に変更
   - より統一感のある見た目に

3. **レスポンシブ対応**
   - モバイル: `gap-4` (16px)
   - デスクトップ: `gap-5` (20px)

## 実装方針

- すべてのグリッドの`gap-6`を`gap-5`に変更
- KPIカードの`gap-4`を`gap-5`に変更
- フォロー中の案件内の`gap-3`を`gap-4`に変更
- カード内パディングを`p-5`から`p-6`に変更
